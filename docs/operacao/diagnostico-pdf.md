# Diagnóstico gratuito — layout do PDF

**Canal:** WhatsApp (arquivo PDF).  
**Prazo:** até **24 horas** após receber o(s) link(s).  
**Escopo:** apontamentos apenas — **sem correções**.

## Pré-requisitos (o que o cliente precisa enviar)

Via WhatsApp (já vem no prefill do site):

| Campo | Para quê |
|-------|----------|
| **Meu nome** | Saber com quem você está falando |
| **Link do site** | Rodar o PageSpeed — **só isso basta** para a análise |

Perfil da Empresa, cidade etc. são **opcionais** e podem vir depois, na proposta/Completo.

## Estrutura do PDF (1–2 páginas)

### Cabeçalho
- Logo Marca Digital
- Título: **Diagnóstico de Performance**
- Cliente / URL analisada
- Data
- Texto fixo: *Documento gratuito. Contém apenas apontamentos; não inclui correções.*

### 1. Resumo em 3 bullets
Ex.: “Mobile 42 · Desktop 78 · CTA WhatsApp pouco visível”

### 2. PageSpeed Insights
- Nota **Mobile** e **Desktop** (números grandes)
- Print **mobile** (prioridade) — ver [prova-pagespeed.md](./prova-pagespeed.md)
- Até **5 problemas** principais (LCP, imagens, JS de terceiro, etc.) em linguagem simples

### 3. Conversão e contato (checklist ✓ / ✗)
- Carrega bem no celular?
- Telefone / WhatsApp fácil de achar?
- Formulário ou CTA claro?
- HTTPS / domínio profissional?

### 4. Perfil da Empresa no Google (se houver link ou acharmos)
- Existe / verificado?
- Fotos, horários, categorias, link do site/WhatsApp
- 2–3 melhorias objetivas

### 5. Próximo passo
- Plano sugerido (Landing / Institucional / Completo) **sem preço obrigatório** ou com “a partir de” da tabela pública
- CTA: “Se quiser, monto a proposta formal”

### Rodapé
- CNPJ, e-mail, WhatsApp
- *Diagnóstico válido como referência na data da análise; scores podem mudar.*

## Ferramentas sugeridas
- [PageSpeed Insights](https://pagespeed.web.dev/) — Mobile + Desktop
- Canva / Google Docs “Salvar como PDF” / Notion export — o importante é layout limpo e legível no celular
